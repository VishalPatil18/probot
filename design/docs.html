Everything you need to build, deploy, and embed your ProBot — your personal AI digital recruiter.

Introduction

ProBot turns your résumé, LinkedIn, and portfolio into a personal AI chatbot that answers recruiter questions 24/7. It's fully open source and runs on your own LLM API key — use the hosted version or self-host. Under the hood it uses a Retrieval-Augmented Generation (RAG) pipeline: your data is chunked, embedded, and stored in a private vector namespace, then the most relevant chunks are retrieved to ground every answer.

New here? Jump to the Quick start to get a live bot in two minutes.

Quick start

Three steps from sign-up to a shareable bot:

  • Sign up with email or OAuth (Google, GitHub, LinkedIn) and claim your username — it becomes your bot URL.
  • Add your model key — Claude, Gemini, DeepSeek, or GPT. Stored locally, never tracked.
  • Upload your résumé PDF, paste a LinkedIn/portfolio URL, or type a bio in the Bot Factory.
  • Deploy — share probot.com/u/your-name or drop the embed snippet on your site.
# Your bot is live at:
https://probot.com/u/vishal

Creating a bot

The Bot Factory is a 4-step wizard: Identity (name, headline, photo), Knowledge base (PDF / URL / text), Personality (tone preset, theme color, custom instructions), and Deploy. Supported knowledge sources:

  • PDF — up to 5 files, max 10MB each
  • URL — LinkedIn profile or portfolio site
  • Free text — up to 50,000 characters

Models & API keys

ProBot doesn't ship its own LLM — you bring your own API key and choose the model that fits your budget and quality needs. Supported providers:

  • Anthropic — Claude Opus 4, Sonnet 4, Haiku 4
  • Google — Gemini 2.5 Pro, Flash, Flash-Lite
  • DeepSeek — DeepSeek-V3, DeepSeek-R1
  • OpenAI — GPT-4o, GPT-4o mini, o3-mini
lock

Your key is stored locally and never tracked by ProBot. It's saved to your browser/self-hosted config and used only to call the provider directly — it never passes through or is logged by ProBot's servers.

Set it in Settings → AI model & key, the Bot Factory wizard, or via config when self-hosting:

// probot.config.json — never committed, never sent
{
  "provider": "anthropic",      // anthropic | google | deepseek | openai
  "model": "claude-opus-4",
  "apiKey": "sk-ant-...",     // read from local env / secure store
  "storage": "local",
  "telemetry": false
}

Prefer not to hard-code it? Point apiKey at an environment variable (e.g. process.env.ANTHROPIC_API_KEY) and keep secrets out of your repo entirely.

Embed the widget

Add a floating chat bubble to any website with one line. The widget is Shadow-DOM isolated (no CSS conflicts) and under 50KB gzipped.

<!-- Paste before </body> -->
<script src="https://probot.com/widget.js"
        data-bot-id="vishal"></script>

Optional attributes:

AttributeDefaultDescription
data-bot-id—Required. Your bot's ID/username.
data-positionbottom-rightBubble placement.
data-color#3461d6Theme color (hex).

Direct link & signature

Beyond the widget, share your bot via your direct URL, an email-signature badge, or your LinkedIn "Featured" / "Contact info" section.

REST API

The public chat endpoint powers both the widget and direct link. Authentication is not required for chat; dashboard endpoints require a session token.

POST/api/chat/:botId
// request
{
  "message": "Does Vishal have RAG experience?",
  "session_id": "uuid-v4"
}

// 200 OK
{
  "reply": "Yes — he built a production RAG chatbot...",
  "conversation_id": "uuid"
}

Other endpoints (session required):

  • GET /api/bots/:botId/analytics — engagement summary
  • GET /api/bots/:botId/leads — captured leads
  • POST /api/bots/:botId/knowledge — ingest new data

Rate limits

Sensible defaults protect your bot — and your own LLM credits — from abuse. When self-hosting, every value is configurable in probot.config.json.

LimitDefaultConfig key
Per recruiter / min10perMinute
Per bot / day200perDay
Max message length8,000 charsmaxChars

A 429 response returns { "error": "rate_limit" } or { "error": "daily_limit" }, and the widget shows a friendly "contact directly" fallback.

Security model

Every bot inherits a layered defense stack adapted from the VAi engine:

  • Input: 40+ prompt-injection patterns, unicode homoglyph normalization, zero-width stripping, 8,000-char limit.
  • Prompt: immutable identity lock, context-only constraint, prompt-protection, override resistance.
  • Output: system-marker detection, JSON/credential scanning, 1,500-char cap.
  • Isolation: per-bot vector namespaces & tenant-scoped queries.
  • Credentials: your LLM API key is stored locally and never transmitted to or logged by ProBot.
shield

Your bot will never reveal its system prompt or invent facts not in your data — it points recruiters to you instead.
