---
title: Dashboard analytics API (v1)
description: "The /api/v1/bot/{conversations,leads} endpoints the probot-self-hosted npm package calls when you link your dashboard for analytics."
---

The `probot-self-hosted` widget only talks to the platform when you supply
a `dashboard.token` - and even then, only to record analytics. Two
endpoints; both authenticated with the bot token minted in the dashboard:

```
Authorization: Bearer pbt_<64 hex>
```

A missing or malformed header returns `401 missing_bot_token`; an unknown or
revoked token returns `401 invalid_bot_token`. The token resolves to exactly
one bot, so no `botId` is ever passed in the request.

<Info>
  This contract is pinned to **v1**. Breaking changes will ship as `/api/v2`
  with a deprecation window, so a deployed webapp can rely on a stable shape.
</Info>

## POST /api/v1/bot/conversations

Persist a chat turn's transcript so it appears in the owner's dashboard.
UPSERTs on `(bot, sessionId)`, so resending the same session coalesces.

```json
// request
{
  "sessionId": "uuid",
  "messages": [
    { "role": "user", "content": "…" },
    { "role": "assistant", "content": "…" }
  ]
}

// response
{ "conversationId": "uuid" }
```

## POST /api/v1/bot/leads

Capture a recruiter lead. Idempotent on `(bot, conversationId, email)`.

```json
// request
{ "email": "recruiter@acme.com", "conversationId": "uuid (optional)", "contextSummary": "… (optional)" }

// response
{ "lead": { "id": "…", "email": "…", "capturedAt": "…" }, "deduped": false }
```

A duplicate returns `200` with `"deduped": true`; a fresh capture returns
`201`.

## What's not exposed

Self-hosted bots configure themselves in your codebase, so there is no
platform-side `config` or `knowledge` endpoint - unlike the earlier runtime
model, the platform never serves persona or retrieval to a self-hosted bot.
This keeps the surface auditable: two endpoints, both write-only, both
scoped to a single bot.
