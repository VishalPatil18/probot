## Beta - June 2026

The Beta version of ProBot shipped in seven stages. Everything below is live in production today.

### Stage 1 - Foundation & Bot Creation

- Next.js 14 App Router scaffolding with TypeScript strict, Tailwind, Drizzle ORM
- PostgreSQL schema (`users` + `bots`) with UUID PKs and proper FK cascades
- NextAuth email/password authentication with JWT session strategy
- Login + register UI (split-panel design)
- Multi-provider LLM client abstraction: Anthropic, OpenAI, Azure OpenAI adapters
- BYO key transport via `x-llm-api-key` header (never in JSON body, never logged)
- Browser key store (`localStorage` Beta default; later migrated to encrypted IndexedDB in Stage 7)
- 5-step Bot Factory wizard (identity → knowledge → personality → AI model → deploy)
- `POST /api/bots` upsert with one-bot-per-user model
- Chat UI with markdown rendering, auto-expanding textarea, suggested questions
- `POST /api/chat/[botId]` with input + output sanitization and 2-tier rate limit

### Stage 2 - Data Ingestion Pipeline

- PDF text extraction via `pdf-parse` with magic-byte verification
- Token-aware chunking via `tiktoken` (cl100k_base, 750/100 with overlap)
- `knowledge_base` table with per-source replace semantics
- `bots.context_text` reassembly from chunks ordered by `(source_name, chunk_index)`
- Per-bot `context_token_cap` (default 12,000) with Advanced disclosure in wizard

### Stage 3 - RAG & Vector Search

- pgvector extension with 1536-dim embedding column
- OpenAI `text-embedding-3-large` truncated via Matryoshka representation
- Optional BYO embedding key (`x-embedding-api-key` header) - bots without a key fall back to full-context
- Cosine similarity retrieval with top-K=3 chunks injected into system prompt
- Graceful fallback to full-context when retrieval fails

### Stage 4 - Public Chat & Onboarding

- Public `/u/[username]/chat` route (no-auth, recruiter-facing)
- Username slug enforcement at registration via regex + reserved-slug set
- Onboarding flow for OAuth / magic-link users to pick a real username
- Default avatar selection (deterministic animal icons by username)

### Stage 5 - Embeddable Widget

- Single-file embeddable widget (`/widget.js`, ~8 KB minified)
- Shadow DOM isolation to prevent host-page CSS conflicts
- CORS-enabled chat API for cross-origin widget calls
- Per-bot theme color (hex, default `#7c5cff`) flowing through widget + signature badge

### Stage 6 - Dashboard, Analytics & Lead Capture

- Conversation persistence: `conversations` + `messages` tables with UPSERT on `(bot_id, session_id)`
- In-chat lead capture after N messages with `leads` table
- In-app notifications with `notifications` table + 30-second poll
- Dashboard home: conversations chart, recent leads, bot share card
- 5-tab settings page: Account / Bot configuration / Knowledge base / Security & privacy / AI model & key
- Notification bell + unread badge in topbar

### Stage 7 - Open-Source Hardening, Compliance & Launch

**Phase 1 - Auth hardening**

- `allowDangerousEmailAccountLinking: false` on GitHub + Google providers
- Email verification gate on credentials login
- Full password reset flow (hashed tokens, 1h TTL, single-use) with `/forgot-password` + `/reset-password` pages
- Email verification tokens (24h TTL) sent on credentials registration

**Phase 2 - Bot enhancements**

- `custom_instructions` field (max 2000 chars) wired into wizard Step 3 + dashboard settings + system prompt builder
- Bot draft / publish flow: bots default to `is_active=false` with a signed HMAC `preview_token`
- Per-bot configurable rate limits with dashboard editor and chat-route enforcement
- Private preview URL (`/u/[username]/chat?preview=<token>`) for draft bots with `robots: { index: false }`

**Phase 3 - Hybrid managed key storage**

- AES-256-GCM envelope encryption with per-bot DEK + env-stored KEK
- `encrypted_llm_keys` table (one row per bot) + `decrypt_audit_log` table (30-day retention)
- `POST/DELETE /api/bots/[botId]/llm-key` for managed key store + revoke
- `GET /api/bots/[botId]/llm-key/audit` returning last 30 days of decrypt events
- `PATCH /api/users/me/llm-prefs` backing the dashboard provider / model switcher
- Live `AIModelKeyTab` with provider switcher, key status badge, audit log, revoke button
- Salted IP hashing for audit-log requester identification
- `redactSensitive` helper for logger payload sanitization

**Phase 4 - Provider polish + reliability**

- Real Google Gemini adapter (`@google/generative-ai`, `gemini-2.5-flash` default)
- DeepSeek removed from forward-looking docs and the provider registry
- In-process per-provider circuit breaker (closed / open / half-open, 5-failure threshold, 30s cooldown)
- Graceful AI fallback: 200 response with `fallback: "circuit_open"` when the breaker is open
- All four providers enabled in the bot factory (Anthropic, OpenAI, Azure, Google Gemini)

**Phase 5 - GDPR compliance**

- `GET /api/users/me/export` streaming JSON bundle (strips `hashedPassword`, `previewToken`, vector embeddings)
- `POST /api/users/me/delete` with 7-day grace period and `deletion_requests` table (email + username snapshot)
- `POST /api/users/me/undo-deletion` public route (token IS auth) + `/(auth)/undo-deletion` landing page
- Daily Vercel Cron at `/api/cron/purge-deleted-accounts` with `CRON_SECRET` auth, fail-closed
- `DeleteAccountModal` + `DeleteBotModal` (GitHub-style typed-identifier + typed-phrase confirmation)
- `DELETE /api/bots/[botId]` (immediate, no grace) wired into bot config tab Danger Zone
- Deletion-initiated + deletion-complete email templates with undo link
- Privacy + Terms copy updated for 7-day grace + undo + 30-day outer bound

**Phase 6 - Upload + key store hardening**

- Malware-scan module with magic-byte + extension + MIME + executable signature blocklist + EICAR detection
- Wired into PDF upload route before `pdf-parse`
- Secure key store with IndexedDB + non-extractable Web Crypto AES-256-GCM master key
- `llm-key-store` + `embedding-key-store` rewritten async on top of the secure key store with one-time localStorage migration
- All three callers (`ChatWindow`, `BotFactoryForm`, `AIModelKeyTab`) migrated to the new async API

**Phase 7 - Launch prep**

- KEK rotation CLI (`npm run kek:rotate` with `--dry-run`); per-row idempotent; re-wraps DEK only
- CI key-leak grep guard (`npm run check:key-leaks`) with per-file allow-list
- Landing page copy updated for the hybrid managed / self-host model
- About page `#hybrid` section with side-by-side managed vs self-hosted cards + operator-trust caveat
- `/self-hosting` docs page (requirements, deploy steps, key-storage choice, KEK rotation runbook, cron secret, honest caveats)
- README rewritten: hybrid hero, updated architecture table, two-panel BYO-key flow diagram, KEK rotation section, Stage 8+ roadmap

### Beta stats

- **12 Drizzle migrations** (0000 → 0012)
- **803 tests** across **87 files**, all green
- **263 source files** scanned by the key-leak guard, 0 violations
- **4 LLM providers** live: Anthropic, OpenAI, Azure OpenAI, Google Gemini
- TypeScript strict, zero `any` in application code, Next.js production build green

Detailed stage breakdowns will land in [the changelog](.) as each stage ships.

### Stage 1 - Branding & Copy Cleanup ✅ (2026-06-21)

- Renamed the product surface from "AI Recruiter" to "AI Assistant" across the landing page, auth screens, and chat UI
- Migrated every `probot.com` reference to the `pro-bot.dev` domain (app code, tests, env example)
- Refreshed the login/signup hero with the post-Beta positioning: free & open source, four LLM providers, encrypted keys, no telemetry
- Removed internal Beta build-plan vocabulary (stage/phase references) from source-code comments so the codebase reads cleanly for new contributors

### Stage 2 - Auth UX & Bug-fix Sprint ✅ (2026-06-21)

- Show/hide password toggle on every password field (login, register, reset)
- "Keep me signed in" option on login - remembered sessions last 30 days, otherwise expire after a day
- Live username & email availability check on signup, so collisions surface before you submit
- Forgot-password is now an in-place modal opened from the login screen
- Aligned the Google / GitHub / Magic Link sign-in buttons
- Sign-out now confirms inline in the sidebar instead of a full-screen dialog

### Stage 3 - Account & Settings Hardening ✅ (2026-06-21)

- Edit your full name and username from Settings → Account, with a live "username taken" check
- Change your password (current-password verification + show/hide toggle)
- Upload a profile photo (JPG/PNG/WebP, 2 MB) that shows on your public chat header
- Redesigned the bot theme-color control: a single color circle that opens a swatch grid + custom color picker

### Stage 4 - Bot Factory & Dashboard Polish ✅ (2026-06-22)

- Give your bot its own profile picture in the Bot Factory (defaults to the ProBot icon); it shows on your public chat page and the embeddable widget
- Pick your bot's theme color while creating it
- One-tap trash icon to remove a queued PDF
- A failed PDF no longer derails bot creation: the rest of your knowledge still saves, and you can retry just the file that failed
- "Share your bot" embed snippets now render as dark, copyable code blocks

### Stage 5 - Sidebar, Notifications & Empty-State Polish ✅ (2026-06-22)

- Cleaner sidebar for brand-new accounts: just "Create bot" and your account, no dead-end links
- Account settings now open without needing a bot first
- Click your name/avatar in the sidebar to jump straight to account settings
- Quick docs link next to the notification bell
- Opt in to an email whenever a new lead is captured (toggle in the notification panel)
- A dismissible banner lets you know when our Terms & Privacy change

### Stage 6 - Marketing & Trust Pages ✅ (2026-06-22)

- New "Why ProBot" page (`/why-pro-bot`): an honest, factual comparison vs generic closed chatbot platforms
- New "Hire me" page (`/hire-me`): about the creator, skills, and how to get in touch
- New "Roadmap" page (`/roadmap`): what's shipped and what's next, with a "Suggest a feature" link to GitHub Discussions
- "Watch demo" button on the landing page opens a borderless video modal (shows a "coming soon" poster until the demo video is published)
- Header and footer now link the new pages; docs links point at `pro-bot.dev/docs`
- Added `sitemap.xml` covering the public marketing pages

### Stage 7 - SEO, Docs & Discoverability ✅ (2026-06-22)

- Site-wide SEO: shared metadata layer with canonical URLs, Open Graph + Twitter cards on every public page, and a metadata base URL
- Dynamic social-card images generated at the edge via `next/og` (no static assets to maintain)
- `Organization` + `SoftwareApplication` JSON-LD structured data on the landing page
- `robots.txt` (allows public pages, keeps the dashboard + API out of the index) alongside the existing `sitemap.xml`
- Documentation reorganized: new Concepts/Guides/Self-hosting/Decisions structure, with new pages for managed vs self-hosted, custom instructions, managed key storage, account export & deletion, and an Embed & share guide
- Architecture page gains diagrams for the managed-vs-self-hosted key flow, envelope encryption, and the deletion lifecycle
- New Architecture Decision Records (ADRs) for the hybrid key model, the deletion grace window, and the circuit-breaker design

### Stage 8 - Performance, Scale & Operational Polish 🟡 (2026-06-22)

- Rate limiter and provider circuit breaker can now share state via Upstash Redis, so several simultaneously-warm serverless instances enforce one accurate limit and an outage that trips one instance protects them all
- Zero-config by default: when Upstash isn't configured, both fall back to fast per-process in-memory state (unchanged behavior, no new cost)
- Operational alerting seam: a structured `circuit_open` event fires at the start of a provider outage, with a Sentry-ready hook (redacted log sink by default)
- Faster account-data export: child rows are bucketed by bot in a single pass instead of a repeated scan
- Much lighter pages: replaced the ~3.86 MB Material Symbols icon font (loaded via a render-blocking stylesheet) with inline SVG icons, and dropped legacy-browser JS polyfills - faster first paint and far less data on the landing page, the public chat, and the dashboard
- Partial stage: production performance measurement, load testing, and live error-alerting wiring are operational/native; some smaller hardening items remain planned
