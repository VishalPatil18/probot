The embed widget is shipped. For a quick overview of sharing options, see
[Embed & share](/embed-share); this page is the full configuration reference.

## What it looks like

A single `<script>` tag on any HTML page:

```html
<script
  src="https://pro-bot.dev/widget.js"
  data-bot-id="00000000-0000-0000-0000-000000000000"
  async
></script>
```

The script:

1. Injects a floating chat bubble in the bottom-right corner.
2. Opens an iframe-isolated chat surface when clicked.
3. Posts messages to the bot owner's hosted `POST /api/chat/[botId]` endpoint.

## Why iframe isolation

The embedding site (someone else's portfolio, blog, or LinkedIn-clone) **must not** be able to read the chat surface's state. Iframe isolation gives us:

- A separate JS realm - the host page can't read `chatHistory` or replies.
- A separate cookie jar - host-page cookies don't ride along.
- A separate `localStorage` - but: the bot owner's BYO key lives on the **owner's browser**, not the recruiter's. The recruiter never sees a key.

## Hosting

The bundle will be built as a standalone Vite/esbuild output and uploaded to AWS S3, fronted by CloudFront. Both fit AWS's **Always Free** tier, keeping with ProBot's [zero-cost policy](https://github.com/vishalpatil18/probot/blob/main/CLAUDE.md#7-strict-zero-cost-policy).

## Configuration

| Attribute       | Required | Meaning                                                 |
| --------------- | -------- | ------------------------------------------------------- |
| `data-bot`      | yes      | The bot UUID (visible in your dashboard after save)     |
| `data-theme`    | no       | `"light"` (default) or `"dark"`                         |
| `data-position` | no       | `"bottom-right"` (default), `"bottom-left"`, `"inline"` |

## Security considerations (planned design notes)

- **CSP-friendly:** the bundle uses no inline scripts; sites with strict CSPs only need to allow the CloudFront origin.
- **No third-party tracking:** the widget pings only `probot.vercel.app/api/chat/…` and CloudFront. No analytics SDK, no Sentry on the recruiter side.
- **Rate limit applies:** the same per-bot 2-tier limiter that protects `/api/chat/[botId]` covers embed-widget traffic.

## What's not in scope for Stage 5

- Themed CSS overrides for the bubble. We'll ship light/dark + a sane default; per-site theming comes later.
- Voice / audio. Text only.
- Resume PDF preview attachment. Tracked separately under Stage 2.
